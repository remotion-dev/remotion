# interpolateRotate()<AvailableFrom v="4.0.472" />

Allows you to map a range of values to CSS rotate values using angle units.

## API

Takes three to four arguments:

1. The input value.
2. The range of values that you expect the input to assume.
3. The range of output rotate values that you want the input to map to.
4. Options object. Optional.

`inputRange` and `outputRange` must have the same length and at least one rotate value.
With one rotate value, `interpolateRotate()` always returns the only output value.

Rotate values may use CSS angle units: `deg`, `rad`, `grad`, or `turn`.
Mixed units are supported, for example `0deg`, `3.14rad`, `100grad`, and `0.5turn`.

## Returns

A CSS rotate value using `deg` units.

## Example

```tsx twoslash
import {interpolateRotate, useCurrentFrame} from 'remotion';

const frame = useCurrentFrame();

const rotate = interpolateRotate(frame, [0, 30], ['0deg', '90deg']);
```

This returns `45deg` at frame `15`.

You can use the value with the CSS `rotate` property:

```tsx twoslash
import {interpolateRotate, useCurrentFrame} from 'remotion';

export const MyComposition: React.FC = () => {
  const frame = useCurrentFrame();
  const rotate = interpolateRotate(frame, [0, 30], ['0deg', '90deg']);

  return <div style={{rotate}}>Hello</div>;
};
```

## Supported units

The `outputRange` may contain `deg`, `rad`, `grad`, and `turn` values.
Values are converted to degrees before interpolation, and `interpolateRotate()` returns a `deg` value.

Other CSS units such as `px`, `%`, or unitless values will throw an error.

## Options

The options are the same as [`interpolate()`](/docs/interpolate):

- [`extrapolateLeft`](/docs/interpolate#extrapolateleft)
- [`extrapolateRight`](/docs/interpolate#extrapolateright)
- [`easing`](/docs/interpolate#easing)
- [`posterize`](/docs/interpolate#posterize)
