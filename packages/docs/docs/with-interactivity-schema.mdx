# withInteractivitySchema()<AvailableFrom v="4.0.479" />

Wraps a React component so Remotion Studio can expose its props as timeline controls.

Use it when you build a component that should behave like [`<Sequence>`](/docs/sequence), [`<Solid>`](/docs/solid) or [`<CanvasImage>`](/docs/canvasimage): The component appears in the timeline, can expose editable controls and can receive saved values from the Studio.

## Example

```tsx twoslash title="Circle.tsx"
import {forwardRef} from 'react';
import {
  Sequence,
  type SequenceControls,
  type InteractivitySchema,
  withInteractivitySchema,
} from 'remotion';

type CircleProps = {
  readonly radius?: number;
  readonly color?: string;
  readonly durationInFrames?: number;
  readonly from?: number;
  readonly name?: string;
};

const circleSchema = {
  radius: {
    type: 'number',
    min: 1,
    step: 1,
    default: 80,
    description: 'Radius',
    hiddenFromList: false,
  },
  color: {
    type: 'color',
    default: '#0b84ff',
    description: 'Color',
  },
} as const satisfies InteractivitySchema;

const CircleInner = forwardRef<
  HTMLDivElement,
  CircleProps & {
    readonly _experimentalControls: SequenceControls | undefined;
  }
>(
  (
    {
      radius = 80,
      color = '#0b84ff',
      durationInFrames,
      from,
      name,
      _experimentalControls,
    },
    ref,
  ) => {
    return (
      <Sequence
        layout="none"
        durationInFrames={durationInFrames}
        from={from}
        name={name ?? '<Circle>'}
        _experimentalControls={_experimentalControls}
      >
        <div
          ref={ref}
          style={{
            width: radius * 2,
            height: radius * 2,
            borderRadius: '50%',
            backgroundColor: color,
          }}
        />
      </Sequence>
    );
  },
);

export const Circle = withInteractivitySchema({
  Component: CircleInner,
  componentIdentity: 'com.example.Circle',
  schema: circleSchema,
  supportsEffects: false,
});
```

The exported `Circle` component only accepts `CircleProps`. The `_experimentalControls` prop is passed by `withInteractivitySchema()` to the inner component and should be forwarded to the [`<Sequence>`](/docs/sequence) that represents the component in the timeline.

## API

```ts twoslash
import {withInteractivitySchema} from 'remotion';
```

### `Component`

The component to wrap.

It must accept its public props plus `_experimentalControls`.

### `_experimentalControls`

The control state created by `withInteractivitySchema()`.

Forward it to the `<Sequence>` that registers the component in the timeline.

### `schema`

An [`InteractivitySchema`](/docs/interactivity-schema) that describes which props are editable in Remotion Studio.

### `componentIdentity`

A stable string that identifies the component type.

Use reverse-DNS style names such as `"com.example.Circle"`. Pass `null` if the component should not be identified for source updates.

### `supportsEffects`

Set to `true` if this component supports an `effects` prop. Otherwise, set it to `false`.

### Return value

Returns a component with the public props of `Component`.

## Behavior

During rendering, in the Player and in read-only Studio sessions, `withInteractivitySchema()` renders the component with the original props.

In editable Studio sessions, it reads the current prop values, applies timeline overrides and passes the merged props to the inner component.
