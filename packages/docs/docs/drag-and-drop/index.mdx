## One drag item

[`makeDragData()`](/docs/drag-and-drop/make-drag-data) returns a MIME type and its serialized payload together:

```tsx twoslash
import {makeDragData} from '@remotion/drag-and-drop';

const dragData = makeDragData({
  type: 'composition',
  compositionId: 'MyVideo',
  compositionFile: 'src/Root.tsx',
  width: 1920,
  height: 1080,
  durationInFrames: 150,
});

export const DraggableComposition = () => {
  return (
    <div
      draggable
      onDragStart={(event) => {
        event.dataTransfer.setData(dragData.mimeType, dragData.payload);
      }}
    />
  );
};
```

This creates one `DataTransfer` item.
Its MIME type identifies the drag type and contains small preview metadata such as dimensions and duration.
Its value contains the canonical JSON payload.

Use [`getDragPreviewMetadata()`](/docs/drag-and-drop/get-drag-preview-metadata) during `dragover`, when the browser exposes MIME types but protects payload values.
Use [`parseDragData()`](/docs/drag-and-drop/parse-drag-data) after the item is dropped to validate and parse both parts together.
