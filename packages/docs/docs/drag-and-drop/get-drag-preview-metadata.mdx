# getDragPreviewMetadata()<AvailableFrom v="4.0.498" />

Finds and parses Remotion preview metadata in an array-like list of MIME types.

```tsx twoslash title="DropArea.tsx"
import {getDragPreviewMetadata} from '@remotion/drag-and-drop';

export const DropArea = () => {
  return (
    <div
      onDragOver={(event) => {
        event.preventDefault();
        const metadata = getDragPreviewMetadata(event.dataTransfer.types);

        if (metadata?.type === 'composition') {
          console.log(metadata.width, metadata.height);
        }
      }}
    />
  );
};
```

Returns `DragPreviewMetadataWithMimeType` for the first valid Remotion drag MIME type, otherwise `null`.
This function only reads `DataTransfer.types`; it does not call `DataTransfer.getData()`, which is unavailable while the browser's drag data store is protected.

Treat this metadata as display-only.
Call [`parseDragData()`](/docs/drag-and-drop/parse-drag-data) when the item is dropped to validate the MIME metadata and canonical payload together.

## Return value

### `type`

The drag kind: `asset`, `component`, `composition`, `effect`, `element`, or `sfx`.

### `mimeType`

The exact MIME type containing the preview metadata.

### `width` and `height`

Preview dimensions when they are available for the drag kind.

### `durationInFrames`

The preview duration for a composition or element, when provided.

### `durationInSeconds`

The preview duration for an asset, when provided.
