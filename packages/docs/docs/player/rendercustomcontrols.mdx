---
title: renderCustomControls
---

_available from v4.1.0_

Allows you to add custom controls to the player, must return a valid React element.

## API

```tsx title="MyApp.tsx"
import {Player, RenderCustomControls} from '@remotion/player';
import {useCallback} from 'react';

const MyVideo = () => {
  return <div>My Video</div>;
};
// ---cut---
export const App: React.FC = () => {
  const renderCustomControls: RenderCustomControls = useCallback(
    ({playing, toggle}) => {
      return (
        <button onClick={toggle}>
          {playing ? 'Pause' : 'Play'}
        </button>
      );
    },
    []
  );

  return (
    <Player
      component={MyVideo}
      durationInFrames={120}
      compositionWidth={1920}
      compositionHeight={1080}
      fps={30}
      renderCustomControls={renderCustomControls}
    />
  );
};
```

The function you pass to `renderCustomControls` receives an object with the following properties:

## Properties

### `playing`

Whether the video is currently playing.

### `play`

A function to start playing the video.

### `pause`

A function to pause the video.

### `toggle`

A function to toggle between play and pause.

### `frame`

The current frame number (0-indexed).

### `durationInFrames`

The total number of frames in the composition.

### `seek`

A function to seek to a specific frame.

```tsx
seek(100); // Seeks to frame 100
```

### `fps`

The frames per second of the composition.

### `isFullscreen`

Whether the player is currently in fullscreen mode.

### `volume`

The current volume (between 0 and 1).

### `setVolume`

A function to set the volume.

```tsx
setVolume(0.5); // Set volume to 50%
```

## Notes

Custom controls are rendered between the main playback controls (play, seek, volume) and the fullscreen button.

Wrap your render function in `useCallback` to avoid unnecessary re-renders.

If your controls use local state, include that state in the `useCallback` dependency array.
